# AmiorAI v40.0.1 — audit Python Embedded et gestion VRAM TTS

Date : 11 juillet 2026

## Correction d’installation

Sous Windows, aucun Python système n’est désormais nécessaire pour la synthèse vocale :

- `tts_server/python_chatterbox` télécharge et utilise Python Embedded 3.11.9 ;
- `tts_server/python_qwen` télécharge et utilise Python Embedded 3.12.10 ;
- le Python principal `python_embed` d’AmiorAI reste indépendant ;
- aucun PATH Windows n’est modifié ;
- les anciens dossiers `venv`, `venv_chatterbox` et `venv_qwen` ne sont plus sélectionnés sous Windows.

Les deux moteurs restent séparés car leurs dépendances ML ne sont pas compatibles. Les poids des modèles ne sont toujours pas redistribués dans l’archive.

## Gestion VRAM retenue

Le réglage `tts_vram_offload_enabled`, activé par défaut, coordonne LM Studio, ComfyUI et le TTS CUDA avec le verrou VRAM existant.

### Avant une conversation ou une tâche LM Studio

1. AmiorAI vérifie si le serveur TTS utilise CUDA.
2. Le processus TTS est arrêté.
3. AmiorAI attend que le serveur local ne réponde plus.
4. ComfyUI libère ensuite ses modèles s’il est inactif.
5. LM Studio peut charger le modèle demandé.

### Avant une génération d’image

1. Le processus TTS CUDA est arrêté.
2. Les modèles AmiorAI présents dans LM Studio sont déchargés.
3. ComfyUI prend la VRAM pour la génération.

### Avant une lecture vocale

1. AmiorAI décharge ses modèles LM Studio.
2. Il attend la fin d’une éventuelle génération ComfyUI.
3. ComfyUI libère ses modèles.
4. Le runtime vocal sélectionné est lancé automatiquement.
5. Le WAV final est renvoyé au navigateur ; sa lecture ne dépend plus du serveur TTS.

L’arrêt du processus est volontaire : il garantit mieux qu’un simple `empty_cache()` la restitution complète du contexte CUDA et des allocations PyTorch. Le TTS CPU n’est pas arrêté, car il n’occupe pas de VRAM.

## Serveur lancé manuellement

Le serveur TTS expose maintenant `POST /shutdown`, limité à l’interface locale. Cela permet à AmiorAI de libérer la VRAM même lorsque le serveur a été démarré manuellement et n’est pas directement enfant du processus principal.

## Interface et migration

- Nouveau choix : **Libérer la VRAM entre les moteurs**.
- Valeur par défaut : oui.
- Les réglages existants reçoivent automatiquement cette valeur lors de l’ouverture de la base.
- Les voix et transcriptions des personnages sont conservées.
- Le catalogue de traduction contient 991 clés synchronisées en FR/EN/ES/DE.

## Limite du test

Les téléchargements PyTorch et les inférences Chatterbox/Qwen n’ont pas été exécutés dans l’environnement de fabrication, qui ne dispose pas de la RTX cible. La première installation et la mesure réelle de VRAM doivent être validées sur la machine Windows équipée de la RTX 5080.
