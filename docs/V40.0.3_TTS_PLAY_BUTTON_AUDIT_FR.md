# Audit correctif AmiorAI v40.0.3 — bouton de lecture TTS

## Défaut constaté

Le bouton manuel de lecture était construit uniquement lorsque la variable JavaScript `ttsEnabled` était vraie au rendu de la conversation. Après activation dans Réglages, la sauvegarde backend réussissait mais l’état local n’était pas synchronisé. La conversation déjà ouverte restait donc sans bouton jusqu’au redémarrage ou à un nouveau chargement complet.

## Correctifs

- synchronisation immédiate de `ttsEnabled` et `ttsAutoplay` après sauvegarde ;
- nouveau rendu automatique de la conversation active lorsque l’état vocal change ;
- bouton explicite `▶ Écouter` sur tous les messages assistant ;
- bouton également présent sur les messages accompagnés d’une image ;
- ajout du même bouton dans l’interface mobile ;
- bouton visible lorsque le TTS est désactivé, avec redirection vers Réglages > Voix ;
- traductions FR/EN/ES/DE et invalidation du cache frontend.
