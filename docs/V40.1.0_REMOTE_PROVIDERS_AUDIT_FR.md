# Audit de mise à jour — AmiorAI v40.1.0

Base utilisée : **AmiorAI v40.0.9 source**.

## Périmètre intégré

- Choix indépendant du fournisseur LLM et du fournisseur d’image.
- LM Studio local, API compatible OpenAI, Runpod Serverless vLLM et Runpod Pod vLLM.
- ComfyUI local, ComfyUI distant, Runpod Serverless ComfyUI et Runpod Pod ComfyUI.
- TTS conservé exclusivement en local.
- Démarrage automatique des Pods au premier appel.
- Arrêt automatique après 15 minutes d’inactivité par défaut.
- Minuteurs indépendants pour le Pod LLM et le Pod image.
- Aucun arrêt pendant une tâche suivie par AmiorAI.
- Commandes Start/Stop manuelles depuis les réglages.
- Clés distantes exclues de SQLite et confiées au gestionnaire de secrets du système.
- Contrôles cloud réservés au PC hôte, même en mode LAN.
- Aucun déchargement inutile de LM Studio ou du TTS lors d’un calcul d’image distant.
- Quatre kits de déploiement documentés dans `runpod_templates`.

## Confidentialité

Le mode local reste inchangé. En mode distant, AmiorAI ne transmet que les données nécessaires au fournisseur sélectionné. Les réglages affichent explicitement qu’un prompt LLM distant peut inclure le contexte et les extraits de mémoire nécessaires, tandis qu’une génération d’image distante transmet le workflow, le prompt et les références éventuelles.

## Sécurité et limites

- Aucune clé n’est renvoyée en clair par `/api/settings`.
- Les routes de secret et de pilotage Pod sont refusées aux clients LAN.
- La fermeture normale tente d’arrêter les Pods configurés lorsque l’arrêt automatique est actif.
- Un crash brutal, une panne ou une coupure réseau peut empêcher l’ordre d’arrêt : le tableau de bord Runpod reste l’autorité finale.
- AmiorAI ne crée aucun compte, ne vend aucun crédit et ne reçoit aucun paiement fournisseur.

## Vérifications automatisées

- compilation Python complète ;
- validation syntaxique JavaScript ;
- validation JSON des traductions et presets ;
- absence d’identifiants HTML dupliqués ;
- tests simulés des API OpenAI-compatible et ComfyUI Serverless ;
- tests de normalisation des fournisseurs ;
- test garantissant que les secrets ne sont pas écrits dans SQLite ;
- test de l’arrêt automatique d’un Pod inactif.

Les tests distants utilisent des serveurs simulés et ne consomment aucun crédit Runpod réel.
