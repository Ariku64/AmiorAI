# Audit du patch v40.0.8 — Personnages et scénarios partageables

## Objectif

Permettre l’échange public de personnages et de scénarios sans exposer les données personnelles accumulées localement.

## Formats ajoutés

- `.amiorchar` : archive ZIP identifiée par un manifeste `amiorai.character` contenant la fiche publique, l’avatar principal et les portraits d’humeur disponibles.
- `.amiorscenario` : archive ZIP identifiée par un manifeste `amiorai.scenario` contenant les données narratives du scénario.

## Données volontairement exclues du paquet personnage

- conversations et messages ;
- mémoires courtes, longues et structurées ;
- statistiques d’affection, confiance, énergie, curiosité et stress ;
- galerie et journal ;
- échantillon vocal et transcription vocale ;
- réglages, chemins locaux, tokens API et identifiants de base de données.

Le paquet conserve les champs nécessaires à la personnalité et à la régénération visuelle : description physique, prompt image, tags verrouillés, token d’identité Krea et option de verrouillage physique.

Les images déjà générées sont copiées dans le paquet. Si un avatar ou un portrait d’humeur manque, son absence ne bloque pas l’export : le destinataire pourra le générer avec sa propre installation ComfyUI et ses modèles locaux. Les textes saisis dans la fiche (personnalité, scénario, message d’accueil et prompts) sont publics dans le paquet et doivent être relus avant diffusion.

## Sécurité d’import

- limite de 128 Mio pour l’archive et 192 Mio après décompression ;
- maximum de 64 entrées ;
- blocage des chemins absolus et des séquences `..` ;
- refus des archives chiffrées ;
- images limitées à 32 Mio et signatures PNG/JPEG/WebP vérifiées ;
- création systématique de nouveaux identifiants afin de ne jamais écraser un personnage ou scénario existant.

## Compatibilité

- les anciens fichiers JSON de personnage restent importables ;
- les scénarios JSON simples restent importables ;
- l’ancien endpoint d’export JSON est conservé pour compatibilité, mais l’interface utilise désormais le paquet public sécurisé.
