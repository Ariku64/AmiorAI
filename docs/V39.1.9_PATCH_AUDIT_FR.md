# AmiorAI v39.1.9 — Patch ergonomie des Réglages

## Objectif

Éviter qu’un utilisateur modifie une section très éloignée du bouton final, oublie de sauvegarder, puis pense que l’application ne fonctionne pas.

## Changements

### Boutons locaux

Un bouton **Enregistrer cette section** est placé après les sections longues qui utilisent la sauvegarde générale :

- moteur de langage ;
- modèle utilitaire ;
- synthèse vocale TTS ;
- dictée Whisper ;
- connexion ComfyUI ;
- gestion de la VRAM.

Ces boutons réutilisent la même route et le même format que la sauvegarde générale : aucune nouvelle source de vérité n’a été créée.

### Retour visuel

Après une modification, la section affiche :

- `Modifications non enregistrées` ;
- `Enregistrement…` pendant l’appel ;
- `✓ Section enregistrée` après succès ;
- un message d’erreur local si la sauvegarde échoue.

Le bouton final reste disponible sous le nom **Enregistrer tous les réglages**.

### Sections particulières

- Persona conserve son bouton dédié, nécessaire pour enregistrer également l’image de persona.
- Les champs persona signalent maintenant les modifications non enregistrées.
- Apparence et langue indiquent qu’elles sont appliquées et sauvegardées immédiatement.
- Correction du bouton Français qui utilisait par erreur `data-lang="en"`.

### Traductions

Ajout synchronisé de 7 clés en FR, EN, ES et DE dans :

- les quatre fichiers JSON ;
- `resources/i18n/translations_master.xlsx`.
