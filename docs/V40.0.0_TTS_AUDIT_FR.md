# AmiorAI v40.0.0 — audit de la refonte vocale

Date : 11 juillet 2026

## Décision

Le moteur XTTS/Coqui de la v39 a été retiré du chemin actif et de l’installation. La v40 utilise Chatterbox Multilingual V3 par défaut et propose Qwen3-TTS 0.6B Base comme moteur expérimental facultatif.

## Architecture retenue

- `tts_server/venv_chatterbox` : moteur principal.
- `tts_server/venv_qwen` : moteur expérimental.
- Port local commun : `127.0.0.1:8810`.
- Le serveur est relancé automatiquement lorsque le moteur configuré change.
- Une requête attend désormais la fin réelle du chargement du modèle afin d’éviter les erreurs HTTP 503 au premier usage.
- L’ancien `tts_server/venv` n’est jamais sélectionné par la v40.

Cette isolation est nécessaire car les deux projets imposent des générations différentes de bibliothèques ML, notamment `transformers`.

## Compatibilité des données

- Les fichiers de voix déjà associés aux personnages sont conservés.
- La base reçoit uniquement une colonne optionnelle `voice_transcript`.
- Les utilisateurs Chatterbox n’ont rien à retranscrire.
- Qwen peut utiliser la transcription exacte pour améliorer le clonage, ou fonctionner sans elle en mode empreinte vocale.

## Solidité de lecture

- Nettoyage du Markdown, des URL et des blocs de code avant synthèse.
- Découpage par phrases plutôt que par tranches arbitraires.
- Normalisation des références vers mono 24 kHz.
- Fusion des segments avec silences courts.
- Contrôles d’expressivité, de fidélité et de température pour Chatterbox.
- Contrôle de vitesse réalisé après synthèse pour garder un protocole identique entre moteurs.

## Installation et licences

Sous Windows, les environnements vocaux exigent un Python 3.11 ou 3.12 système distinct du Python embarqué de l’application. Les poids des modèles ne sont pas redistribués dans l’archive. Les installateurs utilisent des environnements séparés et les sources officielles indiquées dans `THIRD_PARTY_NOTICES.md`. La licence logicielle ne remplace pas le consentement de la personne dont la voix est utilisée ; cette règle figure désormais dans l’avertissement juridique et dans l’interface.

## Contrôles de livraison

- compilation syntaxique de tous les fichiers Python ;
- validation JSON des traductions et workflows ;
- contrôle syntaxique JavaScript et shell ;
- recherche des anciens chemins actifs XTTS/Coqui ;
- contrôle des versions et des marqueurs d’acceptation juridique ;
- migration réelle d’une base v39.1.9 vers v40 avec conservation du personnage et de sa voix ;
- création et mise à jour réelles de personnages avec transcription vocale ;
- validation des paramètres TTS, des signatures audio importées et des réponses WAV.

## Limite du test

L’environnement de fabrication ne possède ni GPU NVIDIA ni poids Chatterbox/Qwen téléchargés. Le démarrage réel des modèles et la qualité audio doivent donc être validés sur la machine cible après installation.
