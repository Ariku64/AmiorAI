# Audit AmiorAI v40.0.7 — ComfyUI tiers uniquement

Date : 12 juillet 2026

## Résultat

L’archive v40.0.6 ne contenait déjà aucun dossier ComfyUI, aucun poids de modèle et aucun installateur ComfyUI embarqué.

En revanche, AmiorAI conservait encore un gestionnaire de processus capable de :

- rechercher le Python d’une installation ComfyUI ;
- lancer `main.py` en arrière-plan ;
- redémarrer, arrêter ou tuer le processus lancé ;
- conserver des réglages de chemin, arguments et délai de démarrage.

Ces fonctions ont été retirées en v40.0.7.

## Architecture retenue

AmiorAI utilise désormais exclusivement une instance ComfyUI tierce :

1. l’utilisateur installe et démarre ComfyUI séparément ;
2. AmiorAI se connecte à son API HTTP locale ;
3. aucune commande de processus ComfyUI n’est exécutée par AmiorAI.

Fonctions conservées via l’API :

- soumission et suivi des générations ;
- lecture de la queue et de `/system_stats` ;
- détection des modèles, LoRA et nœuds ;
- demande `/free` pour libérer la VRAM ;
- diagnostics de connexion.

## Compatibilité

Les anciens réglages `comfy_path`, `comfy_python`, `comfy_autolaunch`, `comfy_extra_args` et `comfy_start_timeout` sont supprimés automatiquement de la base de données au démarrage.
