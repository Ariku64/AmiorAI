# Audit correctif AmiorAI v40.0.2 — installation Chatterbox

## Défaut reproduit

Un téléchargement ou une installation interrompue pouvait laisser le dossier
`tts_server/python_chatterbox` avec `python.exe`, mais sans le paquet Python
`chatterbox-tts`. La présence du runtime suffisait alors à lancer le serveur, qui
passait ensuite en erreur avec `No module named 'chatterbox'`.

## Correctifs

- reconfiguration systématique de `python311._pth` et de `Lib/site-packages` ;
- réinstallation de pip lorsqu'il manque dans un runtime partiel ;
- installation du wheel officiel `chatterbox-tts==0.1.7` sans remplacer le PyTorch
  CUDA 12.8 destiné aux cartes RTX récentes ;
- seconde tentative propre et sans cache si l'import échoue ;
- création du marqueur `.installed` uniquement après l'import réel de
  `ChatterboxMultilingualTTS` ;
- contrôle préventif du module par AmiorAI avant le lancement du serveur ;
- script utilisateur `repair_chatterbox.bat` ;
- journal pip conservé dans `tts_server/install_chatterbox_pip.log`.

## Migration depuis v40.0.1

Il n'est pas nécessaire de supprimer `python_chatterbox`. Fermer AmiorAI puis
lancer `tts_server/repair_chatterbox.bat`. L'installateur conserve le runtime et
les caches de modèles, puis ne répare que les éléments absents ou invalides.
