LM Studio VRAM management

          
Before image generation, AmiorAI can unload from LM Studio only the models it configured itself (chat model and, if the utility model points to the same server, the utility model too), then reload only the model needed by the next text request. Other models loaded in LM Studio are never touched.

          
          
          
          
          
          
Prevents LM Studio from keeping both models in VRAM and pushing the chat model to CPU.


          

            LM Studio reload delays
            
Increase if LM Studio does answer correctly but AmiorAI reports a false error after model switching.

            

              

              

              

              

              

              

              

            

            
          


          

          

            
            
          

        

        

          
          
        

      


      

        
Modèle utilitaire (optionnel)

        
A dedicated instruction-following model can improve structured JSON, image prompts, character creation, scenarios and memory summaries.

        
        
        

          

            

        

        
The utility model uses the same LM Studio server URL and API key as the conversation model.


        
        
Disabled by default: if the utility model is enabled but unreachable, structured tasks fail with a clear message instead of silently using the roleplay model for JSON. Enable this only if you prefer a degraded result to a failure.
